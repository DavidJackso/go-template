# internal/service

Бизнес-логика. Зависит от `domain.UserRepository` через интерфейс — не от конкретного репозитория.

Что здесь происходит:
- валидация входных данных
- хэширование паролей (bcrypt)
- маппинг ошибок репозитория в `apierrors`
- проверка пароля при логине

Не знает про HTTP, статус-коды и JSON. Возвращает `*entity.User` или `*apierrors.APIError`.
